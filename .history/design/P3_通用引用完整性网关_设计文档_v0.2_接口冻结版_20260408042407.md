P3：通用引用完整性网关（v0.2 接口冻结版）
1. 定位（冻结）

P3 是主线能力，属于 logic_fingerprint，负责：

验证回答中的引用是否真实存在，并且属于当前 ContextPool

P2：语义支持（soft check）
P3：引用归属（hard check）
2. 输入接口（冻结）
2.1 Host → P3 请求模型
@dataclass
class HostP3Request:
    request_id: str
    llm_output: str
    context_pool: "ContextPool"
    decision_type: str
    action_risk: str
    metadata: dict[str, Any] | None = None
2.2 ContextPool（核心边界）
@dataclass
class ContextMember:
    member_id: str          # 唯一ID（必须稳定）
    source_id: str          # 文档来源
    content: str            # 内容
    title: str | None = None
    metadata: dict[str, Any] = field(default_factory=dict)

@dataclass
class ContextPool:
    pool_id: str
    members: list[ContextMember]
    retrieval_meta: dict[str, Any]

👉 硬约束：

所有引用必须来自 members.member_id
不允许“模糊引用”（比如“某文档说”）
3. 中间结构（冻结）
3.1 引用解析结果
@dataclass
class ReferenceClaim:
    raw_text: str

    position: int
3.2 解析后引用
@dataclass
class ResolvedReference:

@dataclass
class UnresolvedReference:
    raw_text: str
    reason: str  # parse_failed / id_not_found
4. 输出接口（冻结）
4.1 P3 输出模型

class P3Result:
    plugin_name: str  # "reference_integrity_gateway"

    # 枚举见下

5. 状态枚举（冻结）
clean
partial
unresolved
outside_pool
fabricated
含义：
状态	含义
clean	所有引用合法
partial	部分合法
unresolved	无法解析
outside_pool	不在上下文池
fabricated	伪造引用

13. 不允许做的事（强约束）

P3 不允许：

❌ 做语义判断（那是 P2）
❌ 做行业规则（那是 P4）
❌ 做重试逻辑
❌ 修改 LLM 输出
14. 一句话总结（最终冻结）

P3 = 引用归属校验器（只回答一件事：这个引用是不是来自当前上下文池）
