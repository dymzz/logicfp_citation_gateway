# logicfp-citation-gateway 设计文档

## 1. 项目概述

**项目名称**: logicfp-citation-gateway  
**功能定位**: 引用网关，连接上层应用与本地 `logicfp-kb` 知识库，处理引用和上下文管理

## 2. 核心功能

- **知识库查询**: 调用 `logicfp-kb` 的 query/answer/chat 接口
- **引用管理**: 管理代码片段引用，追踪来源和版本
- **上下文维护**: 以文件形式存储会话、引用记录
- **网关聚合**: 统一暴露接口给上层应用

## 3. 架构设计

### 3.1 核心模块

```
src/
├── gateway.py              # 主网关入口，暴露统一接口
├── citation/               # 引用管理核心
│   ├── models.py          # 数据模型（Citation, Context, etc）
│   └── storage.py         # 文件存储（JSON/YAML）
└── config.py              # 配置管理
```

### 3.2 数据流

```
Application Request
        │
        ▼
    gateway.py (route)
        │
        ├─► logicfp-kb (query/answer/chat)
        │
        ├─► Citation Models
        │
        ▼
   File Storage (session/context)
        │
        ▼
Application Response
```

### 3.3 存储方案

- **会话数据**: `data/sessions/{session_id}.json`
- **引用记录**: `data/citations/{source_id}.json`
- **上下文包**: `data/contexts/{context_id}.json`

## 4. 关键设计决策

1. **直接调用知识库**: 无需中间接口层，直接通过 `uv run --project $LOGICFP_KB_HOME` 调用
2. **文件存储**: 暂不使用数据库，所有状态以 JSON 文件存储在 `data/` 目录
3. **会话隔离**: 每个会话独立文件，便于管理和版本控制
4. **引用追踪**: 每条引用记录来源、时间戳、相关上下文
