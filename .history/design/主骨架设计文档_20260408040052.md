# logicfp-citation-gateway 设计文档

## 1. 项目概述

**项目名称**: logicfp-citation-gateway  
**功能定位**: 引用网关，连接上层应用与本地代码知识库，提供引用管理和上下文池服务

## 2. 核心功能

- **引用管理**: 管理代码片段引用，追踪来源、版本、时间戳
- **上下文池**: 维护引用的上下文集合，支持会话隔离
- **知识库集成**: 与 logicfp-kb 对接，支持查询/问答/对话
- **文件存储**: 完整的 CRUD 操作，JSON 格式，无数据库依赖

## 3. 架构设计

### 3.1 核心模块

```
src/
├── gateway.py              # 主网关入口（CitationGateway）
├── citation/               # 上下文池核心
│   ├── models.py          # 数据模型（Citation, Context, Session）
│   └── storage.py         # 文件存储 CRUD 操作
└── config.py              # 配置管理
```

### 3.2 数据流

```
Application Request
        │
        ▼
    gateway.py (route)
        │
        ├─► logicfp-kb (query/answer/chat)
        │
        ├─► Citation Models
        │
        ▼
   File Storage (session/context)
        │
        ▼
Application Response
```

### 3.3 存储方案

- **会话数据**: `data/sessions/{session_id}.json` - 对话会话
- **上下文池**: `data/contexts/{context_id}.json` - 引用的上下文集合
- **引用记录**: `data/citations/{source}/{citation_id}.json` - 代码片段引用（按来源组织）

## 4. 关键设计决策

1. **上下文池模型**: Context 作为引用的上下文集合，支持灵活的引用组织
2. **会话隔离**: 每个会话独立文件，支持多轮对话，自动维护上下文池
3. **引用追踪**: 每条引用记录来源、时间戳、文件位置等完整信息
4. **文件存储**: 无需数据库，所有数据以 JSON 格式存储，便于版本控制
5. **知识库集成**: 通过 subprocess 调用 logicfp-kb，保持依赖隔离

## 5. API 设计

### 5.1 CitationGateway 接口

```python
class CitationGateway:
    # 会话管理
    def create_session(metadata: Dict[str, Any]) -> Session
    def load_session(session_id: str) -> Optional[Session]
    
    # 上下文管理
    def create_context(session_id: str, metadata: Dict[str, Any]) -> Context
    def add_citation_to_context(context_id: str, citation: Citation) -> None
    
    # 知识库集成（Phase 3+）
    def query_kb(query: str, top_k: int) -> Dict[str, Any]
    def answer_kb(question: str, output_format: str) -> Dict[str, Any]
    def chat_kb(session_id: str, message: str) -> Dict[str, Any]
```

### 5.2 数据模型

#### Citation（引用）
```python
@dataclass
class Citation:
    id: str                           # UUID
    source: str                       # KB 来源（logic_fingerprint 等）
    code_snippet: str                 # 代码片段
    description: Optional[str]        # 描述
    file_path: Optional[str]          # 源文件路径
    line_range: Optional[str]         # 行号范围 (start-end)
    timestamp: str                    # ISO 8601 时间戳
    metadata: Dict[str, Any]          # 自定义元数据
```

#### Context（上下文）
```python
@dataclass
class Context:
    id: str                           # UUID
    session_id: str                   # 关联的会话
    citations: List[Citation]         # 包含的引用列表
    created_at: str                   # 创建时间
    updated_at: str                   # 更新时间
    metadata: Dict[str, Any]          # 自定义元数据
```

#### Session（会话）
```python
@dataclass
class Session:
    id: str                           # UUID
    source: str                       # KB 来源
    created_at: str                   # 创建时间
    updated_at: str                   # 更新时间
    context_ids: List[str]            # 包含的上下文ID
    metadata: Dict[str, Any]          # 自定义元数据
```

## 6. 工作流示例

### 6.1 基础流程：创建会话→上下文→引用

```
1. 应用启动
   └─ gateway = CitationGateway(kb_source="logic_fingerprint")

2. 用户开始对话
   └─ session = gateway.create_session({"user_id": "user123"})

3. 创建对话上下文
   └─ context = gateway.create_context(session.id)

4. 添加代码引用
   └─ citation = Citation(source="logic_fingerprint", code_snippet="...")
      gateway.add_citation_to_context(context.id, citation)

5. 持久化存储
   └─ data/
      ├─ sessions/{session_id}.json
      ├─ contexts/{context_id}.json
      └─ citations/{source}/{citation_id}.json
```

### 6.2 扩展流程：知识库集成（Phase 3+）

```
1. 用户提问
   └─ message = "What does LogicFingerprintMiddleware do?"

2. 调用网关查询
   └─ results = gateway.query_kb(message, top_k=5)

3. 知识库返回匹配内容
   └─ 通过 subprocess 调用 logicfp-kb:
      uv run --project $LOGICFP_KB_HOME kb-query "{message}"

4. 创建引用与上下文
   └─ for result in results:
        citation = Citation(source="logic_fingerprint", ...)
        gateway.add_citation_to_context(context.id, citation)

5. 返回结果
   └─ Application Response
```

## 8. 与 logicfp-kb 的交互

### 8.1 调用方式

直接通过 subprocess 调用知识库（避免直接导入）：

```python
import subprocess
import os

kb_home = os.getenv("LOGICFP_KB_HOME")
result = subprocess.run(
    [
        "uv", "run",
        "--project", kb_home,
        "kb-query",
        "search terms here",
        "--source", "logic_fingerprint",
        "--top-k", "5"
    ],
    capture_output=True,
    text=True
)
output = result.stdout
```

### 8.2 支持的 KB 操作

- `kb-query`: 代码检索（返回 top-k 匹配）
- `kb-answer`: 单轮问答（返回问题 + 证据 + 答案）
- `kb-chat`: 多轮对话（支持 session 持久化）

## 9. 错误处理策略

1. **环境变量缺失**: 检查 `LOGICFP_KB_HOME` 是否设置
2. **文件 I/O 错误**: 确保 `data/` 目录可写
3. **KB 不可用**: 标记 `query_kb` 等方法为 NotImplementedError
4. **无效数据**: 在加载 JSON 时进行 validation

## 10. 扩展性设计

### 10.1 支持多个 KB 来源

```python
# 可扩展支持多个知识库
gateway_lf = CitationGateway(kb_source="logic_fingerprint")
gateway_cred = CitationGateway(kb_source="logicfp_credibility")
```

### 10.2 自定义元数据

每个数据模型都支持 `metadata` 字段，允许灵活扩展：

```python
citation = Citation(
    source="logic_fingerprint",
    code_snippet="...",
    metadata={
        "relevance_score": 0.95,
        "kb_search_query": "middleware fallback",
        "custom_tag": "important"
    }
)
```

### 10.3 未来数据库集成

当需要数据库时，只需实现 `BaseStorage` 接口替换 `FileStorage`：

```python
class BaseStorage(ABC):
    @abstractmethod
    def save_session(self, session: Session) -> None: ...
    @abstractmethod
    def load_session(self, session_id: str) -> Optional[Session]: ...
    # ... 其他方法
```
