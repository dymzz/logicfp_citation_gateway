# P3 总览与目录

## 0.1 文档目的

本文档是 P3（通用引用完整性网关）设计文档集的总览页，用于完成三件事：

1. 给出整套文档的正式目录与阅读顺序；
2. 说明各文档的职责边界，避免重复阅读和交叉误解；
3. 作为后续版本演进、代码实现、Benchmark 编写、回归执行和主程序集成时的统一导航入口。

本文档本身不承担新的规则定义职责。  
P3 的能力边界、状态枚举、输入输出对象、聚合规则、接入时序、回归规范与版本控制，均以下列正式分章文档与附录为准。

---

## 0.2 P3 的定位

P3 的正式定位为：

**通用引用完整性网关（Reference Integrity Gateway）**

P3 只负责判断一件事：

**当前回答中的引用，是否属于当前 ContextPool。**

P3 不负责：

- 语义支持判断；
- 推理合理性判断；
- 真实世界正确性判断；
- 自动修复引用；
- 自动补全引用；
- 最终业务放行决策。

P3 的唯一合法引用边界来源是 **ContextPool**。  
P3 的输出必须进入主程序统一聚合层，由主程序结合 P2 结果生成最终 `DecisionResult`。

## 0.3 文档集使用原则

在阅读和使用整套 P3 文档时，应遵守以下原则：

### 0.3.1 总览页不替代正式分章

本文档只做导航与索引，不替代正式设计章节。  
若本文档与正式分章内容冲突，应以对应正式分章与附录为准。

### 0.3.2 分章负责不同层面的冻结

P3 文档集不是一篇大而全说明书，而是按职责拆分为多个正式文档：

- 总设计文档负责总体定位与接口冻结；
- 约束文档负责红线与禁止项；
- 聚合文档负责 P2 / P3 / 主程序之间的统一出口；
- 接入文档负责宿主接入顺序与调用约束；
- Benchmark / 回归文档负责测试资产标准化；
- 版本文档负责后续演进控制；
- 附录负责术语、枚举、错误码与 contract 收口。

### 0.3.3 代码实现、Benchmark、回归报告都必须可回指文档

后续实现中，任何以下产物都应能明确回指对应文档章节：

- 代码中的状态枚举；
- parser / resolver / evaluator 行为；
- 主程序聚合逻辑；
- Benchmark case 的 expected 字段；
- 回归报告中的失败解释；
- 版本升级说明。

## 0.5 各文档职责说明
### 0.5.1 P3_00_总览与目录

作用：

作为整套文档的导航入口；
固定正式目录；
规定推荐阅读顺序；
说明各文档的角色边界。
### 0.5.2 P3_01_通用引用完整性网关_总设计与接口冻结

作用：

定义 P3 的总体定位；
冻结 P3 的输入输出边界；
明确 P3 的最小实现范围；
作为整套设计的主文档。

适用场景：

设计评审；
新成员快速理解 P3；
主程序接入前确认核心 contract；
后续版本升级时确认是否触碰冻结边界。
### 0.5.3 P3_02_约束与禁止项

作用：

固定 P3 的设计红线；
防止 P3 漂移成“半语义判断器”或“自动修复器”；
明确哪些行为绝不属于 P3。

适用场景：

parser / resolver 设计时检查是否越界；
评审“是不是要加一点智能化”时做边界裁决；
拒绝把 P3 写成第二个 P2。
### 0.5.4 P3_03_主程序聚合规则与DecisionResult_Contract

作用：

规定 P2 与 P3 的聚合优先级；
固定主程序统一出口；
定义 DecisionResult 的最小 contract；
明确 verdict / trigger_codes / reason_codes / error_codes 的统一表达方式。

适用场景：

主程序落地聚合逻辑；
接入层代码实现；
block / warn / fallback / allow 的统一判定。
### 0.5.5 P3_04_最小接入时序与宿主调用约束

作用：

固定宿主接入 P3 的标准时序；
明确调用前置条件；
明确哪些属于宿主输入错误；
固定最小调用方式与禁止项。

适用场景：

宿主系统接入 P3；
对接 HTTP / pipeline / middleware 场景；
校验“P3 是否被放在正确位置”。
### 0.5.6 P3_05_Benchmark与回归样本约束

作用：

规定 P3 benchmark 的目标与边界；
固定样本分类方式；
冻结每类样本的测试重点；
防止 benchmark 失去状态边界约束意义。

适用场景：

建立首版 benchmark 集；
评审样本覆盖是否完整；
回归时检查是否覆盖核心状态路径。
### 0.5.7 P3_06_Benchmark样本编写规范与Case模板

提供统一 case 模板；

适用场景：

### 0.5.8 P3_07_回归执行规范与结果报告模板

固定失败分级口径；

### 0.5.10 P3_A_术语表与冻结枚举总表

## 0.6 推荐阅读顺序

推荐阅读顺序固定为：

P3_01 → P3_02 → P3_03 → P3_04 → P3_05 → P3_06 → P3_07 → P3_08 → P3_A

原因如下：

先看总设计
先确定 P3 是什么，不是什么。
再看约束与禁止项
先把边界钉死，避免后续理解跑偏。
再看主程序聚合
明确 P3 不是最终决策器，而是统一 decision 链的一环。
再看最小接入时序
明确宿主系统如何调用，什么位置接入才正确。
再看 Benchmark 与回归文档
在边界清楚之后，再定义如何验证边界没有被破坏。
最后看版本控制与附录
作为长期维护、升级和收口依据。

## 0.8 文档间的优先级关系

当不同文档出现交叉引用时，优先级按以下顺序处理：

附录 A 的术语、枚举、错误码、对象字段定义
对应主题的正式分章文档
P3_01 主设计文档
P3_00 总览页

也就是说：

术语、状态、trigger、error code 冲突时，以附录 A 为准；
聚合逻辑冲突时，以 P3_03 为准；
宿主接入时序冲突时，以 P3_04 为准；
Benchmark 样本规范冲突时，以 P3_05 ~ P3_07 为准；
本文档只做导航，不单独创造例外规则。
