1. 核心原则（不可破坏）
1.1 单一职责原则（强约束）

P3 只做一件事：引用是否属于当前 ContextPool

P3 的唯一问题是：

这个引用是不是来自当前上下文池？

不允许扩展为：

“这个引用是否合理”
“这个引用是否语义支持”
“这个引用是否真实世界正确”
1.2 ContextPool 是唯一真理来源（强约束）
ContextPool = 唯一合法引用集合

任何引用判断：

✔ 只基于 ContextPool
❌ 不允许访问外部知识
❌ 不允许调用 embedding / LLM
❌ 不允许做语义补全
1.3 结构优先于语义（强约束）

P3 是结构校验，不是语义判断：

类型	是否属于 P3
ID 是否存在	✔
是否属于 pool	✔
内容是否支持结论	❌（P2）
是否合理推理	❌（P2）
2. 明确禁止事项（红线）
2.1 ❌ 禁止做语义判断

以下行为全部禁止：

- 使用 embedding 判断“相似引用”
- 用 LLM 判断“这个引用可能对应哪个文档”
- 自动修正引用 ID
- 模糊匹配（fuzzy match）

👉 否则会发生：

P3 退化为“第二个 P2”，职责崩溃

2.2 ❌ 禁止访问外部系统

P3 不允许：

调用向量数据库
调用检索系统
调用 API
查询数据库补全引用

👉 P3 必须是：

纯内存、纯函数式、零副作用模块

2.3 ❌ 禁止修改 LLM 输出

P3 只能“判断”，不能“修复”：

❌ 不允许：
- 自动补引用
- 自动替换引用
- 自动删除引用
- 自动纠错

👉 否则会破坏：

审计链（audit chain）

2.4 ❌ 禁止引入业务逻辑

P3 不能出现：

法律规则
医疗规则
金融合规
SOP判断

👉 这些属于：

P4（外部插件）
2.5 ❌ 禁止侵入主程序决策

P3 只输出：

status / triggers / reasons

不允许：

❌ 直接返回 verdict（allow/block）
❌ 控制流程分支
❌ 调用 fallback

👉 决策权永远在主程序（P1）

2.6 ❌ 禁止“智能化过度”

禁止加入：

置信度学习
自适应权重
自动调参
AI判断引用合理性

👉 P3 必须是：

确定性规则系统（deterministic）

3. 输入约束（必须满足）
3.1 ContextPool 必须完整
- member_id 必须唯一
- 不允许为空
- 不允许重复

否则：

→ 直接视为系统错误，不进入 P3
3.2 引用格式必须可解析

当前冻结格式：

[ref:<member_id>]

P3 不负责：

兼容各种 citation 风格
自动识别自然语言引用

👉 否则 parser 会失控

4. 输出约束（必须稳定）
4.1 状态枚举不可扩展
clean
partial
unresolved
outside_pool
fabricated

❗禁止新增：

“almost_clean”
“weak_reference”
“maybe_outside”
4.2 triggers 必须可枚举

所有 trigger：

必须在 constants.py 固定定义

❌ 禁止：

动态生成 trigger
拼字符串 trigger
4.3 输出必须可审计

P3Result 必须保证：

- 可复现
- 可解释
- 可追溯
5. 边界划分（最重要）
5.1 P2 vs P3（绝对不能混）
能力	P2	P3
语义支持	✔	❌
引用存在性	❌	✔
推理合理性	✔	❌
ID校验	❌	✔
5.2 P3 vs 主程序（P1）
能力	P3	主程序
判断引用	✔	❌
决策 allow/block	❌	✔
audit输出	❌	✔
错误码映射	❌	✔
5.3 P3 vs P4
能力	P3	P4
通用引用校验	✔	❌
行业规则	❌	✔
审批流程	❌	✔
自动纠错	❌	✔
6. 可扩展点（允许，但必须受控）
6.1 可扩展：引用格式解析器

允许：

parser.py 增加多格式支持

但必须保证：

输出统一 ReferenceClaim
6.2 可扩展：引用类型

允许未来支持：

doc_id
chunk_id
source_id
composite_key

但必须：

最终映射到 ContextMember
6.3 可扩展：metrics

允许增加：

coverage_rate
resolution_rate

但：

不影响核心 status 判定
7. 设计底线（总结）
P3 的本质：
不是“判断对不对”
而是“引用是否合法”
一句话红线：

只要你开始用语义、用AI、用外部知识 —— 你就已经把 P3 写错了

8. 最终冻结声明
P3 = 引用归属网关（Reference Membership Gateway）

输入：LLM Output + ContextPool  
输出：Reference Integrity Status  
约束：纯结构、纯本地、纯确定性
